# 安全规则详解

## 1. 备用金绝对不确认

```
if "备用金" in message:
    改价 → 跳过确认付款 ❗
```

原因：备用金订单是客户预存的资金，不需要实际收款。确认付款会导致重复扣款。

## 2. 保守跳过原则

```
无法匹配任何白名单关键词 → 跳过
```

宁可少处理，不能误处理。原因：
- 留言内容是客户自由文本，不可控
- 噪音远多于指令（叫车、装车、备注等占 80%+）
- 误操作（不该确认的确认了、不该改价的改了）影响资金安全

## 3. 处理上限

单次运行最多处理 **15 个订单**（跳过不计入），原因：
- 防止脚本异常导致批量误操作
- 需要人工确认上一批结果后再继续

## 4. 刷新策略

每处理 5 个订单刷新页面，原因：
- 防止页面状态过期
- 部分操作（确认付款）会改变订单状态，刷新后当前页不再显示

## 5. 禁止操作

- ❌ 不要打开订单详情页
- ❌ 不要修改运费（除非留言明确要求）
- ❌ 不要关闭订单
- ❌ 不要删除订单
- ❌ 不要询问用户（全自动执行）

## 6. 生产环境保护

- 所有按钮点击前用 `data-id` 精确定位
- 操作后等待页面响应（800ms）
- 改价后验证金额（容差 ±0.01）
- 异常时记录日志并回滚弹窗（点取消）
