# 待讨论事项

## 1. PyInstaller 打包成独立 exe

**问题**：当前代码运行需要 Python 3.12+ 环境和 Playwright、requests 等依赖。目标电脑若无 Python 环境则无法运行。

**方案**：
- 使用 PyInstaller 将 `run.py` 或 `gui.py` 打包为独立 exe
- `config.py` 和 `gui.py` 已预留 `sys.frozen` 兼容代码
- 需验证：打包后 Playwright 的 Chromium 浏览器是否需要额外处理

**需要确认**：是否需要打包？目标电脑是否有 Python 环境？

## 2. 自动重登录机制

**问题**：当前 Cookie 过期后，脚本只会写告警并等待手动登录。若无人值守则订单处理停摆。

**方案**：
- 方案 A：检测到登录过期时，弹出浏览器窗口，发送通知（微信/钉钉）请人手动登录
- 方案 B：保存账号密码，自动重新登录（需逆向登录流程，有被封风险）
- 方案 C：短信验证码自动接收（需 Twilio 类服务）

**需要确认**：采用哪个方案？是否有账号密码保存需求？

## 3. 远程告警推送

**问题**：目前异常告警只写入 `data/alerts.txt` 文件。无实时推送，运维需主动查看日志。

**方案**：
- 企业微信 Bot Webhook（推荐，公司已使用企业微信）
- 钉钉 Bot
- 邮件推送

**需要确认**：使用哪个推送渠道？企业微信 Bot Token ？

## 4. 多店铺支持

**问题**：当前配置文件只绑定了阿凡提商贸一个店铺（shop_id: 8016976）。

**方案**：
- 扩展为多店铺配置（config.json 切换或 GUI 选择）
- 各店铺独立 browser profile

**需要确认**：是否有其他店铺需要接入？

## 5. 改名费支持

**问题**：留言中偶尔包含"免运费"需求。目前自动跳过，需人工处理。

**方案**：
- 新增关键词"免运费"，自动将运费设为 0
- 或保持现状（人工处理）

**需要确认**：是否要支持自动免运费？

## 6. API 模式的稳定性

**问题**：API 模式速度快（每单 <500ms），但依赖启小铺 API 的稳定性。若网站更新 API 端点则失效。

**方案**：
- 默认使用混合模式（浏览器提取 + API 操作）
- API 失败时自动回退到 UI 模式
- 定期验证 API 可用性

**需要确认**：API 模式作为默认还是可选？

## 7. 数据一致性校验

**问题**：每日上千订单，偶尔出现改价金额与实际不符（概率极低但影响资金安全）。

**方案**：
- 增加每日对账功能：统计所有改价订单，与目标金额比对
- 异常订单单独标记

**需要确认**：是否有每天核实订单的需求？

## 8. 运行记录持久化

**问题**：当前 `state.json` 只记录最近一次运行结果，不保留历史。

**方案**：
- SQLite 数据库存储每日处理记录
- Web 管理面板查看历史

**需要确认**：是否需要长期历史记录？
