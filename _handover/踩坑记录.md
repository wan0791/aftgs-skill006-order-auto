# 踩坑记录

## 1. 改价弹窗数据污染问题

**问题**：多次改价时，弹窗的 `priceDiff` 会累加，导致服务端数据被污染。

**现象**：弹窗内的 `realPrice` 显示为 `productTotal + freight + 上次 diff + 本次 diff`，而不是预期的 `productTotal + freight + 本次 diff`。

**解决方案**（`browser_automation.py` `modify_price()` 方法）：
1. 打开弹窗后，先读取服务端真实值（productTotal、freight、current_diff）
2. 计算预期 realPrice = productTotal + freight + current_diff
3. 若预期与实际不符，**拒绝操作**并写入告警
4. 差价计算公式改为：`diff = target - productTotal - freight`（不是 `target - current_price`）

**相关代码**：`browser_automation.py` 第 460-518 行

## 2. `net::ERR_ABORTED` 页面导航错误

**问题**：`navigate_to_orders()` 调用 `page.goto()` 时偶尔抛出 `net::ERR_ABORTED`，原因是服务器 URL 重定向中断了导航。

**现象**：
```
Page.goto: net::ERR_ABORTED; maybe frame was detached?
```

**解决方案**（`browser_automation.py` 第 175-177 行）：
- 在 `goto()` 外层包 try-except，忽略导航异常
- 导航后检查 URL 是否为目标页，而不是依赖 `goto()` 的返回值

**相关代码**：`browser_automation.py` 第 171-194 行

## 3. 留言解析样式差异

**问题**：启小铺的留言 HTML 结构与微分销不同，原 CSS 选择器无法定位留言。

**现象**：`extract_orders()` 返回的订单中 `message` 字段为空。

**解决方案**（`browser_automation.py` 第 286-298 行）：
- 优先用 `p.message` CSS 选择器定位留言
- 兜底用 `.buyer_reply` 的 `data-message` 属性
- 适配两种不同的页面结构

## 4. Windows GBK 编码问题

**问题**：Python 默认 GBK 输出时，包含中文字符（尤其是 emoji）的日志无法正常显示。

**现象**：终端报编码错误（`'gbk' codec can't encode character`）。

**解决方案**（`config.py` 第 145-147 行）：
```python
if sys.stdout.encoding != 'utf-8':
    sys.stdout.reconfigure(encoding='utf-8', errors='replace')
```
- 日志输出前做 `encode(decode)` 替换，确保不抛出编码异常

## 5. 运行锁冲突

**问题**：当脚本被计划任务或手动多次启动时，两个实例同时操作订单。

**解决方案**（`config.py` 第 285-292 行）：
- `data/running.lock` 文件锁
- 锁文件在 150 秒内视为有效（硬超时 120 秒 + 30 秒缓冲）
- 脚本退出时自动清理锁文件（`finally` 块）

## 6. 浏览器进程管理

**问题**：需要保持浏览器常驻以维持登录状态，但 Python 进程退出后浏览器不应关闭。

**解决方案**：
- `BrowserAutomation` 通过 CDP 连接已有浏览器，不管理浏览器生命周期
- Chrome 通过 `subprocess.Popen` 启动后 Python 不 wait，退出后浏览器继续运行
- `stop()` 方法只释放 Playwright 连接，不关闭浏览器
- 下次运行通过 CDP 自动重连

## 7. 弹窗二次确认

**问题**：确认付款时，部分浏览器/页面状态会显示二次确认弹窗，部分不会。

**解决方案**（`browser_automation.py` 第 540-566 行）：
- 点击确认后，等待弹窗出现（5000ms timeout）
- 读取弹窗文本，检查是否包含"确认付款"或"是否继续"
- 不匹配时点取消，不盲目确认

## 8. API 模式的 Cookie 共享

**问题**：API 模式需要从浏览器导出 Cookie 到 requests session。

**解决方案**（`api_client.py` `load_cookies_from_browser()`）：
- 通过 Playwright `page.context.cookies()` 获取所有 cookie
- 导入到 `requests.Session` 的 cookie jar 中
- 后续所有 API 请求使用同一 session

## 9. 商品总价提取偏差

**问题**：提取订单数据时，商品总价可能包含运费，导致 API 改价计算结果错误。

**解决方案**（`browser_automation.py` 第 240-248 行）：
```python
# ¥60.00 (含运费 ¥28.00) 格式解析
orderPrice = totalPrice - freight  # 商品总价 = 实际支付 - 运费
```
在 `api_client.py` 中也使用同样的还原逻辑：
```python
diff = target - orderPrice - freight
```
