# 技术架构

## 技术选型

| 组件 | 选型 | 原因 |
|------|------|------|
| 语言 | Python 3.12+ | 开发效率高，跨平台 |
| 浏览器驱动 | Playwright Python 库 | 比 `time.sleep()` 更可靠的 `wait_for_selector()` |
| HTTP 客户端 | requests | 轻量，适合 API 模式 |
| 浏览器连接 | CDP (Chrome DevTools Protocol) | 复用已有 Chrome 进程，保持登录状态 |
| GUI | tkinter（内置标准库） | 零额外依赖，适合 Windows 环境 |
| 调度 | Windows 计划任务 / `--loop` | 灵活选择 |

## 模块划分

### 1. `run.py` — 主入口

```
main()
  ├── 读取命令行参数 (--loop / --dry-run / --api / --single)
  ├── 运行锁检测 (防止并发)
  ├── BrowserAutomation.start() → CDP 连接/启动浏览器
  ├── 可选: ApiClient 初始化 (导出 cookie)
  ├── process_orders() 主循环
  │     ├── navigate_to_orders() → 导航到待付款页
  │     ├── extract_orders() → 提取所有待付款订单
  │     ├── MessageParser.parse() → 解析留言
  │     ├── browser.modify_price() / api.modify_price()
  │     ├── browser.confirm_payment() / api.confirm_payment()
  │     └── 每 5 单刷新页面
  └── 写 state.json + 清理运行锁
```

### 2. `browser_automation.py` — 浏览器自动化

核心模块，通过 Playwright CDP 连接已有 Chrome 进程：

```
BrowserAutomation
  ├── start()              → CDP 连接 / subprocess 启动 Chrome
  ├── navigate_to_orders() → 导航 + 登录检测
  ├── extract_orders()     → JS 提取订单数据 (DOM 解析)
  ├── modify_price()       → 改价流程 (弹窗 → 读服务端值 → 校验 → 填差价 → 提交 → 验证)
  ├── confirm_payment()    → 确认付款 (点击 → 二次确认弹窗 → OK)
  └── verify_before_confirm() → 确认前金额验证
```

**改价流程**（安全关键）：
1. 点击 `.j-modify[data-id="xxx"]` → 打开改价弹窗
2. 从弹窗读取服务端真实值（productTotal / freight / current_diff）
3. **校验公式**: `realPrice == productTotal + freight + current_diff`
4. 计算差价：`diff = target - productTotal - freight`
5. 填入差价 → 确定 → 验证金额

### 3. `message_parser.py` — 留言解析器

白名单匹配策略，只认 4 个关键词：

```
留言 → 空/图标？ → SKIP
     → 含"备用金"？ → 有金额？ → 金额≠当前？ → MODIFY_SKIP
                                 → 金额=当前？ → SKIP
     → 含"改价"？   → 有金额？ → 金额≠当前？ → MODIFY_CONFIRM
                                 → 金额=当前？ → SKIP
     → 含"直接确认"/"确认付款"？ → CONFIRM_ONLY
     → 默认 → SKIP (保守跳过)
```

### 4. `api_client.py` — HTTP API 客户端

基于逆向的 API 端点，直接从浏览器导出 Cookie 后使用：

| API | 端点 | 方法 |
|-----|------|------|
| 改价 | `/Order/cancle_order` | POST |
| 确认付款 | `/Order/comfrimPay` | POST |
| 订单列表 | `/Order/ajax_order_list` | POST |

APIs 需配合浏览器 Cookie 使用（`load_cookies_from_browser()`）。

### 5. `config.py` — 配置与工具

提供统一配置、日志记录器、告警写入、运行锁、订单记录集合。

### 6. `gui.py` — GUI 控制台

tkinter 桌面应用，功能：
- 启动/暂停/停止 按钮
- 实时日志显示
- 订单处理结果列表（带颜色标记）
- 累计统计
- 灵活定时（00:00-06:00 每 10 分钟，其他时段每 2 分钟）

### 7. `init_login.py` — 登录初始化

首次使用脚本：启动浏览器 → 注入 Cookie → 验证登录 → 保存 profile。
