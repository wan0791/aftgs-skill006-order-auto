# 项目概述

## 项目背景

江西阿凡提商贸有限公司是一家食材配送企业，使用「启小铺」微分销平台进行线上接单。每天有大量待付款订单需要处理，主要工作是：
- 根据买家留言改价（修改订单金额）
- 确认付款

此前使用 [[技能005-启小铺订单]]（基于 Playwright MCP，依赖 Claude Code Agent），每次运行消耗约 5K tokens，且需要 Agent 常驻。

## 项目目标

开发一个**独立运行**的订单自动化脚本，脱离 Claude Code Agent，实现：
1. 零 Token 消耗
2. 支持 Windows 计划任务定时运行
3. 运行稳定、容错性强
4. 异常可追溯（日志 + 截图 + 告警文件）

## 使用场景

| 场景 | 操作 |
|------|------|
| 买家留言"改价30" | 改价到 ¥30 + 自动确认付款 |
| 买家留言"备用金50" | 改价到 ¥50，**不确认付款** |
| 买家留言"直接确认" | 当前金额直接确认付款 |
| 无留言 / 噪音 | 跳过（保守原则） |
| 连续 3 单失败 | 自动停止，写告警文件 |

## 运行模式

| 模式 | 命令 | 说明 |
|------|------|------|
| 单次运行 | `python run.py` | 处理完退出 |
| 常驻模式 | `python run.py --loop` | 每 5 分钟一轮 |
| 演习模式 | `python run.py --dry-run` | 只读不写 |
| API 模式 | `python run.py --api` | HTTP 直调（每单 <500ms） |
| GUI 控制台 | `python gui.py` | tkinter 界面，支持启动/暂停/停止 |
| 单订单 | `python run.py --single 40711315` | 只处理指定订单 |

## 控制文件

编辑 `data/control.json` 即可在线切换模式（无需重启）：

```json
{"mode": "active"}    → 正常运行
{"mode": "paused"}    → 暂停
{"mode": "stopped"}   → 停止（常驻进程退出）
{"mode": "dry_run"}   → 演习模式
```
