# 更新日志

## v1.1.0 (2026-07-06)

### 新特性

- **新增独立模块 `delivery_updater.py`**：待发货订单留言修改功能，完全独立于改价系统
- **收货人关键词匹配**：查找收货人含"4号线"（精确匹配，排除14号线/24号线等）的订单
- **留言自动修改**：将匹配订单的留言"叫车"替换为"装车"，并追加"——已自动化处理"标记
- **分页自动翻页**：支持多页订单自动翻页处理
- **定时调度模式**：`--schedule` 参数，按预设时间自动执行（默认凌晨 3:00）

### 安全机制

- 演习模式 `--dry-run`：只预览匹配结果，不实际修改
- 独立控制文件 `data/delivery_control.json`，不与改价系统冲突
- 独立运行锁 `data/delivery_running.lock`
- 独立日志文件 `data/logs/delivery_update_*.log`

### 配置更新

- `config.py` 新增待发货相关配置（DELIVERY_URL、CSS选择器、定时时间等）

### GUI 增强

- **双标签页设计**：改价处理 + 留言修改，独立按钮和日志窗口
- **留言修改面板**：执行修改/演习模式/停止 按钮
- **定时调度界面**：支持 3 个独立时间段设置，启动/停止定时按钮
- **EXE 图标**：自定义 logo（多尺寸 ICO 嵌入）

### 打包部署

- **统一启动入口** `main.py`：整合改价/留言修改/GUI/调度，支持 `--version` 查看版本
- **PyInstaller 打包**：`build.spec` + `build_exe.bat`，输出独立 EXE
- **版本管理**：`version.py` 统一版本号
- 输出文件：`dist/启小铺订单自动化.exe`（~55MB）

## v1.0.0 (2026-05-18)

### 新特性

- **独立 Playwright 库版本**：脱离 Claude Code Agent，零 Token 消耗
- **UI 模式**：通过 Playwright CDP 连接 Chrome，浏览器自动化操作
- **API 模式**：HTTP 直调启小铺 API，每单处理 <500ms
- **混合模式**：浏览器提取订单 + API 操作订单（高效 + 可靠）
- **GUI 控制台**：tkinter 桌面应用，启动/暂停/停止/实时日志
- **留言解析器**：白名单关键词匹配，保守跳过策略

### 安全机制

- 改价后强制 DOM 验证金额（容差 ±0.01）
- 改价前读取服务端真实值，防止数据污染
- 备用金订单改价后不确认付款
- 连续 3 单失败自动停止并写入告警
- 运行锁防止并发执行
- 软超时 10 分钟 + 硬超时 11 分钟
- 单次最多处理 15 单

### 运维设施

- 带时间戳日志系统，跨 0 点自动切换文件
- 超过 2 天的日志自动归档到月份子目录
- 每日汇总统计前一天处理量
- 异常自动截图保存
- `data/control.json` 在线切换运行模式
- `--loop` 常驻模式（每 5 分钟）
- `--dry-run` 演习模式（只读不写）

### 已知限制

- 首次运行需手动登录（browser profile 保存后续自动登录）
- 异常通知目前只写文件，无实时推送
